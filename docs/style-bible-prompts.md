# Style Bible 生成手册（ChatGPT 网页版）

> Style Bible = 风格圣经。所有后续 700+ 张资产都通过 `reference_images` 引用它，决定整个游戏视觉是否统一。
> **这一步建议在 ChatGPT Plus 网页版交互式生成**（GPT Image 2 Instant/Thinking 模式），而不是直接走 API：
> - 网页版可以反复对话微调，比 API 调试快 5-10 倍
> - 满意后另存原图（避免压缩）放到 `assets/_style_bible/` 即可
> - 总预算 ~$10，相当于 ChatGPT Plus 月费的一小部分

---

## 操作流程

1. 打开 https://chatgpt.com，确认右上角是 Plus / Pro / Business
2. 模型选 **GPT-5.5**（或最新的有图像能力的 4o 后继），点击 + 号 → "图像生成模式"
3. 按下方 6 个 prompt **依序贴入**，每段都要等图出完再继续
4. 不满意就用 "更细节地…" / "把头发改成…" 之类的追问微调
5. 满意后右键图 → "保存图像"，按指定文件名保存
6. 全部 6 张完成后，整个 `assets/_style_bible/` 目录提交 git（这是项目最重要的资产之一）

```

- 美术风格：港漫巨匠马荣成《风云》漫画的厚涂质感，
  墨线分明、色彩浓郁、人物五官立体、肌肉线条有力、衣袂飘动有动势

       网络红人脸、迪士尼风、文字水印、签名
```

## 2️⃣ 步惊云表情 8 宫格 — `02_protagonist_face_sheet.png`

```
第二张：步惊云的表情参考表（face sheet）。

构图：
- 1024×1024 正方形画布
- 严格 2×4 网格（4 行 2 列），8 个表情头像
- 每个格子角色都是同一人（步惊云）
- 仅头部 + 颈部，半身像
- 与上一张全身立绘完全相同的脸型、五官比例、发型

8 个表情按从左到右、从上到下顺序：
1. 平静（neutral，默认对话用）
2. 愤怒（angry，眉头紧锁、双目圆睁）
3. 悲痛（grief，眉头上挑、眼眶微红、嘴角下压）
4. 嘲讽（mocking，单边嘴角上挑）
5. 震惊（shocked，眼睛睁大、嘴微张）
6. 冷笑（cold smile，眼神不变但嘴角扬起）
7. 疲惫（exhausted，眼皮微垂、汗珠）
8. 眯眼专注（focused，双眼微眯、有杀意）

要求：
- 所有 8 张严格保持同一人脸（这是参考表的核心价值）
- 每个格子下方留小空间标注表情名（中文）
- 背景纯白
- 同样港漫厚涂质感
```

## 4️⃣ UI 套件示例 — `04_ui_kit_sample.png`

```
第四张：游戏 UI 套件总览（UI kit sample）。

构图：
- 1024×1024 正方形画布
- 一张图内同时展示多种 UI 组件，仿佛是一份"UI 设计稿"
- 浅灰背景便于看清边界

需要展示的组件：

【顶部，按钮区】
- 三个矩形按钮并排，长宽比 3:1
- 分别为 normal / hover / pressed 三态
- 深棕底（#3a2820）+ 金色描边（#c9a961）
- 按钮内文字依次："开始游戏" / "继续游戏" / "退出"
- hover 态边框发光，pressed 态底色加深

【中部，对话框】
- 一个大对话框，长宽比 4:1
- 边框金色花纹（云纹），四角对称
- 底色深色羊皮纸质感
- 左下角小三角指示符
- 内部模拟显示一段对话文本：
  "步惊云：（默然不语，剑眉微皱）"
- 字体端正

【底部左侧，武功图标 4 个】
- 圆形深色底 + 金色描边
- 内含简化武功意象 + 招式名汉字
- 招式名：排云掌 / 风神腿 / 莫名剑法 / 排山倒海

【底部右侧，道具图标 4 个】
- 方形深色底 + 不同品质边框光（白/蓝/紫/橙）
- 物品：回春丹 / 银两 / 火麟剑 / 绝世好剑

整体风格统一：港漫厚涂金属反光，所有汉字必须正确清晰，
不要乱码、不要英文。
```

## 5️⃣ 45° 等距场景示例 — `05_scene_sample_isometric.png`

```
第五张：45° 等距视角场景示例（isometric scene）。

构图：
- 1536×1024 横版
- 严格 45° 等距视角俯视，**绝对不要透视变形**
- 镜头略微居高俯瞰

场景：天下会大殿外的广场

需要包含：
- 远景：大殿建筑（飞檐斗拱、深色木结构、青瓦屋顶）
- 中景：宽阔的石板地面广场
- 前景：两侧立着旗杆，旗帜上书"天"字
- 中央有一个小人物（步惊云）站立，身高占场景高度的 1/12
  （让玩家可以判断游戏内角色站在场景中的尺寸比例）
- 几处兵器架、火盆作为装饰
- 时辰：黄昏，光线偏暖橘色
- 天气：晴，轻微暮色

要求：
- 港漫厚涂质感，墨线明显
- 留出可通行区域，不要被装饰挤满
- 画面边缘可适度暗角
- 色调统一在【场景色组】内
- 不要文字、地名标签
```

## 6️⃣ 战斗姿态参考 — `06_combat_pose_reference.png`

```
第六张：战斗姿态参考图（combat pose reference）。

构图：
- 1024×1024 正方形
- 步惊云施展招式"排云掌"的关键瞬间，半身偏全身
- 视角与游戏内 sprite 保持一致：斜向 45° 略俯视

姿态：
- 双脚一前一后稳稳扎马步
- 双掌平推向前方（朝右下方 45° 方向）
- 上身略前倾，整个动作充满力量感
- 长发与披风向后飘扬（表现出招瞬间的爆发力）

招式特效：
- 双掌前方涌出半透明的"云形"气劲，蓝白色调
- 气劲呈半圆扩散
- 周围空气有轻微扭曲

要求：
- 仍是马荣成漫画风格的厚涂
- 与第一张全身立绘完全相同的脸型 / 服装
- 背景纯白色（这张主要用作 sprite_attack 的参考锚点，
  GPT Image 2 看到这张就知道"战斗时角色应该是这个样子"）
- 不要文字、不要招式名

这张完成后，整套 Style Bible 就全了。
```

## 微调技巧

如果某张没出对，**不要重头开始**，直接在 ChatGPT 对话里追问：

- "脸形再瘦一些，颧骨更突出"
- "披风颜色再深一档，往黑红走"
- "构图把人物往左移一点，留出右侧文字空间"
- "把这张图重新生成一次，但保持脸不变，只改披风颜色"

> ChatGPT 网页版 GPT Image 2 的 Thinking 模式自动支持参考前面已生成的图，是它最大的优势。

## 经验记录

> 用户规则：每次出现问题修复后做经验记录。

- **为什么不用 API 跑 Style Bible**：Style Bible 需要反复对话微调（"再瘦一些"/"换个颜色"），API 模式调试一次成本 $0.21、要重写 prompt，网页版 ChatGPT Plus 月费 $20 内随便迭代，效率高 10 倍
- **保存图时要原图**：ChatGPT 默认 PNG 是已压缩版，右键"保存图像"或"在新标签页打开图像"再保存 = 接近原始质量
- **8 表情图比单张表情更稳**：单张让 GPT 反复生成同一张脸成功率 < 50%，8 宫格在一次生成内强制约束同一人，成功率 > 90%
